核心概念

什麼是數據中台？

餐廳裡，如果每個廚師都自己去市場買菜 → 採購重複、格式不統一、有人買到壞的。
數據中台，就是那個統一採購的倉管。

❌ 沒有中台

前端 → Yahoo Finance API
前端 → 證交所 API
前端 → 財報 API
──────────────
每頁自己處理格式
換資料來源要大改
錯誤處理各自為政

✅ 有數據中台

Yahoo Finance ─┐
證交所 API    ├→ FastAPI → 前端
財報 API      ─┘
──────────────
統一格式 / 統一快取
統一錯誤處理
前端只問一個地方

數據流

資料怎麼進來的？— ETL Pipeline

Extract
抽取
定時從外部資料來源抓取原始資料 證交所上市股價、Yahoo Finance K 線、財務報表數據
Transform
轉換
清洗、統一格式、計算技術指標 補齊缺漏欄位、計算 KD / MACD / 布林通道，清除異常值
Load
載入
寫入資料庫，供數據中台查詢 前端查詢時，資料已是處理好的乾淨資料，不需要即時運算

技術亮點

我們用了什麼技術？解決了什麼？

技術 角色 解決的問題
FastAPI 數據中台 API 核心 統一資料格式與入口
ETL Pipeline (Python) 資料收集 自動化股票資料整合
Dify 開源 AI 平台 AI 分析 無需訓練模型，快速接 LLM
nginx Gateway 路由層 單一對外入口，方便擴展
Docker Compose 部署 一鍵啟動所有服務
React + Vite 前端 快速開發現代化 UI
