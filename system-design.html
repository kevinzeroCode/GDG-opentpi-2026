3. DigiRunner 在這個專案裡做什麼？

DigiRunner 可以理解成「API 閘道」。它可以把外部 API 包成統一入口，也能負責驗證、轉發、管理外部服務。 目前專案保留 DigiRunner 服務與 token 整合能力，同時 FastAPI 已直接處理主要股票資料流程。

DigiRunner 的角色

  • 容器：quantdashboardai-digirunner-1
  • 本機入口：http://localhost:31080
  • 容器內服務：:18080
  • 可作為 TWSE、FinMind 等外部 API 的 Gateway / Proxy。
  • 後端已有取得 DigiRunner token 的能力。

目前實際使用方式

核心查詢多數由 FastAPI service 直接處理，例如 Dify、FinMind、TWSE、Redis 與 PostgreSQL。 DigiRunner 目前更像是系統預留的 API Gateway 能力，之後可以把外部 API 統一收斂到它後面。

簡單說：FastAPI 是現在主要的資料中台；DigiRunner 是 API Gateway 層，適合未來把外部服務集中管理。
路徑 目前角色 一般人理解
/api/... 前端呼叫 FastAPI 網站主要功能入口。
/twse/ nginx / Vite proxy 可導到 DigiRunner 或 TWSE 路徑 可把證交所資料包成統一入口。
/finmind/ nginx / Vite proxy 可導到 DigiRunner 或 FinMind 路徑 可把 FinMind 資料來源放到 Gateway 後面。
/dgrv4/tptoken/oauth/token FastAPI 可向 DigiRunner 取得 gateway token 後端可以用服務帳號登入 DigiRunner。

4. 部署、SSL 與自動更新

GCP VM

專案部署在 GCP Compute Engine。README 記錄的固定 IP 是 104.199.213.119，Docker Compose 在 VM 裡啟動服務。

自己的 HTTPS 憑證

外層 nginx 監聽 443 ssl， 使用 /etc/letsencrypt/live/quant-opentpi.duckdns.org 底下的憑證。

GitHub Actions 部署

push 到 main 後，workflow 會 SSH 到 GCP，pull 最新程式、重建 frontend， 並重啟 data-platform。

正式環境路徑：使用者走 https://quant-opentpi.duckdns.org， nginx 用 SSL 接住，再把請求轉給 frontend container。
